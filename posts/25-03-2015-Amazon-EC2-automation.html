Scenario

I had to setup an automated process where I can start an EC2 instance along with several other steps, which are as follows -

  • Create a Security Group for EC2 instance. This was because I wanted the instance to serve requests only from the ELB.
  • Create a seperate Security Group for ELB. This was because I wanted a custom rule for each ELB.
  • Create an ELastic Load Balancer with required parameters.
  • Most importantly I had to create a autoscale group, to give me automatic scaling, in case.

Setting up EC2 instance with ELB and Autoscaling Group.

Setting up of the instance is easy. We just need to have a specific format of doing things and things work. The following is how I did things -

  • Get Zones/Zone Names that will be used.
  • Create Both Security Groups
  • Create Load Balancer
  • Create AutoScale Group
all_zones = conn_reg.get_all_zones() zoneStrings = [] for zone in all_zones: zoneStrings.append(zone.name) sg1 = conn_reg.create_security_group(sg1_name, 'Skyrelr-%s for ec2'%skyrelr_instance.email) sg = conn_reg.create_security_group(skyrelr_instance.security_group, 'Skyrelr-%s for elb'%skyrelr_instance.email) sg.authorize('tcp', 80, 80, '0.0.0.0/0') lb = conn_elb.create_load_balancer(skyrelr_instance.loadbalancer_name, zoneStrings, settings.ELASTIC_LOAD_BALANCER['connection_forwarding'], security_groups=[sg.id]) lb.configure_health_check(hc) skyrelr_instance.loadbalancer_dns = lb.dns_name # EC2 instance only gets access from ELB conn_reg.authorize_security_group( group_name=sg1.name, src_security_group_name=sg.name, src_security_group_owner_id=sg.owner_id) lc = create_lc(skyrelr_instance.launch_configuration, sg1_name) conn_as.create_launch_configuration(lc) asg = create_autoscalinggrp(skyrelr_instance.auto_scaling_group, skyrelr_instance.loadbalancer_name, zoneStrings, skyrelr_instance.launch_configuration) conn_as.create_auto_scaling_group(asg) return skyrelr_instance, sg
